# Thinking Coach

一个面向产品设计师的思考教练 skill。

它不直接替用户下结论，而是通过提问帮助用户理清问题、看见约束、扩展方案空间，并在设计决策、复盘和评审中做出更清醒的判断。

## 适用场景

- 用户在多个设计方案之间摇摆不定
- 用户一上来就在讲方案，但问题本身还没有被看清
- 用户想复盘项目、梳理成长、提取经验
- 用户说“帮我想想”“我不确定该怎么做”“这个方向对吗”

## 这个 skill 会做什么

- 用提问方式帮用户回到真正的问题
- 从系统视角看设计决策，而不是只看局部界面
- 打开方案空间，避免过早收窄
- 帮用户做清醒的权衡，而不是替他拍板
- 在适合的时候推动最小有效行动

## 工作模式

### 模式一：即时教练

面向具体设计问题，帮助用户完成：

- 问题诊断
- 系统洞察
- 方案空间扩展
- 方案评估
- 行动推进

### 模式二：复盘教练

面向项目回顾，帮助用户完成：

- 决策回溯
- 方法论盘点
- 成长提取

## 安装

### 方式 A：安装 `.skill` 文件

```bash
claude skill install thinking-coach.skill
```

### 方式 B：直接使用源码

将该目录放入 Claude skills 目录，并命名为 `thinking-coach`：

```text
~/.claude/skills/thinking-coach/
├── SKILL.md
├── LICENSE
└── README.md
```

## 适用边界

- 这个 skill 优先服务“帮用户想清楚”，而不是“帮用户立刻产出结论”
- 如果用户明确要求直接给建议，应当退出教练模式，切到直接建议模式
- 它适合做元能力 skill，和系统思维、设计评审类 skill 互补
